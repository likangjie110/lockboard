## 任务2：完善异常处理和用户提示

### 改进内容

#### 1. ⏱️ 超时处理机制

**实现：**
- 添加了 QTimer 超时定时器
- 默认超时 2000ms
- 发送命令时启动，接收响应时停止
- 超时时显示详细的故障排查对话框

**文件修改：**
- `serialmanager.h` - 新增超时相关接口
- `serialmanager.cpp` - 实现超时检测逻辑

#### 2. ✓ 参数验证

**验证项目：**
- 设备地址范围（1-15）
- 锁号/通道号范围（1-12）
- 多选操作至少选择一项
- 串口是否已选择
- 发送数据非空验证

**错误提示：**
- 清晰的参数错误对话框
- 明确指出有效范围
- 在调试控制台记录错误

#### 3. 🔌 串口断线检测

**实现：**
- 检测 ResourceError 错误类型
- 自动触发 portDisconnected 信号
- 显示设备断开警告
- 自动关闭串口并重置UI状态

**断开处理：**
- 关闭串口连接
- 重置连接按钮
- 清空锁状态显示
- 启用端口选择

#### 4. 📊 详细错误分类

**串口错误分类：**
- DeviceNotFoundError → "设备未找到"
- PermissionError → "权限错误，设备可能被其他程序占用"
- WriteError → "写入数据失败"
- ReadError → "读取数据失败"
- ResourceError → "设备资源不可用，可能已断开连接"
- 等等...

**数据解析错误分析：**
- 包头错误 → 提示总线冲突或地址错误
- 校验错误 → 提示线路干扰
- 长度不匹配 → 提示通讯不稳定

#### 5. 📝 命令执行状态反馈

**发送前日志：**
```
[信息] 开锁 - 设备地址:1, 锁号:5
[信息] 开多个锁(0x15) - 设备地址:1, 选中锁: [1, 2, 3], 掩码:0x0007
```

**响应后反馈：**
```
[信息] 设备版本号: V1.0
[信息] 开锁操作 - 锁5: 打开
[信息] 开全部锁操作完成 - 开启数量: 12/12, 状态掩码: 0x0FFF
```

**操作失败提示：**
- 显示详细的失败原因对话框
- 列出可能的问题
- 提供解决建议

#### 6. ⚠️ 特殊命令警告

**持续输出警告：**
```
[警告] 此命令用于继电器、照明灯等持续通电设备，对电控锁使用可能造成损坏
```

**同时开锁说明：**

```
[信息] 说明：此命令用于同一门上安装多个锁的场景，所有锁同时动作
```

## 编译说明

项目使用 Qt 5.15.2 + MinGW 8.1.0，编译步骤：

```powershell
# 设置Qt环境变量
$env:PATH = "D:\QT\5.15.2\mingw81_64\bin;D:\QT\Tools\mingw810_64\bin;$env:PATH"

# 清理旧文件
mingw32-make clean

# 生成Makefile
qmake LockBoardTester.pro

# 编译
mingw32-make

# 可执行文件位置
# release/LockBoardTester.exe
```
