# 异常处理和用户提示改进文档

## 改进日期：2026-06-30

## 改进概述

本次改进全面增强了485锁控板测试工具的异常处理和用户提示功能，提升了软件的健壮性和用户体验。

---

## 一、超时处理机制 ✅

### 1.1 响应超时检测

**改进内容：**
- 在 `SerialManager` 中添加了 `QTimer` 超时定时器
- 默认超时时间：2000毫秒（2秒）
- 可通过 `setResponseTimeout()` 动态调整超时时间

**实现细节：**
```cpp
// serialmanager.h
void setResponseTimeout(int milliseconds);
int responseTimeout() const;
signal void responseTimeout();

// serialmanager.cpp
- 发送数据时启动超时定时器
- 接收到数据时停止超时定时器
- 超时触发时发出 responseTimeout() 信号
```

**用户提示：**
- 超时时显示详细的故障排查指引对话框
- 包含可能原因和建议操作步骤
- 在调试控制台记录超时详情

## 二、参数验证 ✅

### 2.1 设备地址验证

**验证规则：**
- 有效范围：1-15
- 对应硬件拨码开关4位二进制编码
- 0x00（全OFF）视为无效地址

**错误提示：**
```
参数错误
设备地址 X 超出有效范围

有效范围：1-15
请确认设备拨码开关设置
```

### 2.2 锁号/通道号验证

**验证规则：**
- 有效范围：1-12
- 对应12路锁/通道接口

**错误提示：**
```
参数错误
锁号/通道号 X 超出有效范围

有效范围：1-12
```

### 2.3 多选操作验证

**验证规则：**
- 至少选择一个锁/通道
- 位掩码不能为0

**信息提示：**
```
提示
请至少选择一个锁

在多锁操作标签页中勾选需要操作的锁
```

## 三、串口错误详细分类 ✅

### 3.1 增强的错误提示

原来：仅显示系统错误字符串
现在：根据错误类型提供具体说明

**错误分类：**

| 错误类型 | 错误代码 | 友好提示 |
|---------|---------|---------|
| 设备未找到 | DeviceNotFoundError | "设备未找到" |
| 权限错误 | PermissionError | "权限错误，设备可能被其他程序占用" |
| 无法打开 | OpenError | "无法打开设备" |
| 写入失败 | WriteError | "写入数据失败" |
| 读取失败 | ReadError | "读取数据失败" |
| 资源不可用 | ResourceError | "设备资源不可用，可能已断开连接" |
| 不支持的操作 | UnsupportedOperationError | "不支持的操作" |
| 设备未打开 | NotOpenError | "设备未打开" |

### 3.2 串口断开检测

**新增功能：**
- 检测到 `ResourceError` 时自动触发 `portDisconnected()` 信号
- 显示设备断开警告对话框
- 自动关闭串口并重置UI状态
- 清空锁状态显示

**断开提示对话框：**
```
设备断开
串口设备已断开连接

可能原因：
• USB线缆松动或拔出
• RS485转换器断电
• 设备驱动异常

请重新连接设备后再试
```

## 四、数据解析失败处理 ✅

### 4.1 详细的解析错误分析

**改进内容：**
- 解析失败时不仅记录错误，还提供故障分析提示
- 根据错误类型给出针对性建议

**错误类型与建议：**

| 错误类型 | 原因分析 | 建议 |
|---------|---------|------|
| 包头错误 | 数据格式不符合协议 | "可能存在总线冲突或设备地址错误" |
| 校验错误 | 数据传输过程损坏 | "数据传输可能存在干扰，建议检查RS485线路" |
| 长度不匹配 | 数据帧不完整 | "接收到不完整的数据帧，可能是通讯不稳定" |

### 4.2 无效数据包头处理

**改进内容：**
- 丢弃无效数据时记录丢弃字节数
- 在调试控制台显示详细信息

**日志示例：**
```
[错误] 接收到无效数据包头，已丢弃5字节
[错误] 丢弃包头前的3字节无效数据
```

## 十、数据发送前验证 ✅

### 10.1 空数据检查

**验证内容：**
```cpp
if (data.isEmpty()) {
    emit errorOccurred("发送数据为空");
    return false;
}
```

### 10.2 发送字节数验证

**验证内容：**
```cpp
if (bytesWritten != data.size()) {
    emit errorOccurred(QString("数据未完全发送: 期望%1字节，实际%2字节")
                      .arg(data.size()).arg(bytesWritten));
    return false;
}
```
